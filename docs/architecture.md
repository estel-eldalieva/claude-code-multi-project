# 4레이어 설정 아키텍처

## 개요

Claude Code의 설정 파일들을 4개 레이어로 구조화하여, 각 레이어가 적절한 범위에서만 로드되도록 합니다.

```
범위: 넓음 ──────────────────────────────────── 좁음

레이어 1         레이어 2         레이어 3         레이어 4
글로벌           Rules           메모리           프로젝트
CLAUDE.md        *.md            memory/*.md      CLAUDE.md
(모든 세션)      (파일 매칭 시)   (CT 세션)        (해당 세션만)
```

## 레이어 1: 글로벌 CLAUDE.md

**경로**: `~/.claude/CLAUDE.md`
**로드 시점**: 모든 세션 시작 시 자동

모든 프로젝트에 공통으로 적용되는 규칙:
- 응답 언어
- 코딩 공통 규칙 (Type Hinting, 12-Factor 등)
- 모델/리소스 전략 (난이도별 모델 배분)
- SSH/서버 접근 규칙
- 작업 진행 원칙

**핵심**: 여기에 넣는 내용은 **모든 세션에 토큰 비용**이 들므로, 진짜 글로벌한 것만.

## 레이어 2: Rules

**경로**: `~/.claude/rules/*.md`
**로드 시점**: 파일 패턴 매칭 시 자동

각 파일의 frontmatter `paths:` 패턴에 맞는 파일을 열/편집할 때만 로드:

```yaml
---
paths:
  - "**/*.py"
---
```

언어/프레임워크별 세부 규칙을 여기에 분리하면 Python 작업할 때만 Python 규칙이 로드됩니다.

**장점**: 불필요한 규칙이 컨텍스트를 차지하지 않음.

## 레이어 3: 메모리

**경로**: `~/.claude/projects/<encoded-path>/memory/*.md`
**로드 시점**: 해당 프로젝트 경로의 세션에서 자동

control-tower처럼 여러 프로젝트를 관리하는 세션에서 사용:
- 프로젝트 레지스트리
- 피드백 (실수에서 배운 규칙)
- 세션 관리 체계

**주의**: MEMORY.md 인덱스를 통해 모든 메모리 파일이 로드되므로 총 크기 관리 필요.

## 레이어 4: 프로젝트 CLAUDE.md

**경로**: `<project>/CLAUDE.md`
**로드 시점**: 해당 프로젝트 디렉토리에서 세션 시작 시

프로젝트 고유 정보:
- Tech Stack
- 디렉토리 구조
- 명령어
- 프로젝트 규칙/제약

## 레이어 간 원칙

1. **중복 금지**: 상위 레이어에 있는 내용을 하위에서 반복하지 않음
2. **참조**: 하위 레이어에서 상위를 참조 ("글로벌 CLAUDE.md 참조" 식으로)
3. **구체성**: 상위일수록 일반적, 하위일수록 구체적
4. **단일 원천**: 팩트는 한 파일에만 기술, 나머지는 참조
