# Global Settings

## Language
- 항상 한국어로 응답 (영어 응답 명시 지시 시 제외)
- 커밋 메시지, 주석도 한국어

## Coding Rules (공통)
- 기존 기능/에러 핸들링 절대 삭제 금지
- 코드는 전체 실행 가능한 형태로 작성 (import, config 포함)
- 파일 상단에 버전, 타임스탬프, 주요 변경사항 주석
- 설정은 로직과 분리 (12-Factor)
- 명시적 Type Hinting, 매직 넘버 금지
- 변경 완료 시 변경사항 요약 제공
- 언어/프레임워크별 세부 규칙: `~/.claude/rules/` 참조

## Project Overview 관리
- 의미 있는 변경 작업 완료 시, 프로젝트 루트의 `PROJECT_OVERVIEW.md`를 현행화할 것
- 포맷: 프로젝트명, 한줄 설명, 시스템 구조, 기술 스택, 현재 상태, 최근 변경사항

## Model & Resource Strategy

난이도에 따라 모델과 리소스를 적절히 배분할 것:

| 난이도 | 계획 | 실행 | 예시 |
|--------|------|------|------|
| 단순 | 불필요 | haiku 단발 | 오타 수정, 파일 이동, 간단 조회 |
| 보통 | 본 세션에서 간략히 | sonnet/haiku 에이전트 | 단일 파일 기능 추가, 버그 수정 |
| 복잡 | opus로 계획 수립 | 단계별 모델 지정 + 병렬 에이전트 | 멀티파일 기능, 아키텍처 변경 |
| 대규모 | opus 계획 + 브레인스토밍 | 팀 구성 + 스킬 워크플로우 | 신규 프로젝트, 크로스프로젝트 작업 |

## 작업 진행 원칙
- 같은 접근법으로 3회 이상 실패하면 멈추고 다른 방법을 제안하거나 유저에게 확인
- SSH/빌드/배포 등 장시간 명령이 타임아웃되면 재시도하지 말고 상황 보고
- 해결이 안 되면 빠르게 유저에게 알리고 판단을 구할 것
- 작업 범위가 커질 때는 먼저 유저에게 확인 후 진행
